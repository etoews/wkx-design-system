WKX design system · four options

Four visual languages for the operational interfaces of wkx-platform, each carrying the same surface: fleet, vitals, alarms, deploys, parameters, and the monthly account. Same facts, four points of view.

  • 1 · Red Ledger · light, serif, narrative

    Wing Kong Exchange is a trading house, so the platform keeps a trading ledger: vermilion-ruled accounts, jade chop stamps for state, the budget settled like a monthly book.

  • 2 · Harbour Watch · dark, dense, signal-driven

    Two berths, cloud and home, watched at night. Status is a navigation light with documented meaning: starboard green underway, anchor amber holding, port red fault.

  • 3 · Namespace v0 · light, monospace, interactive

    The first take on the path grammar as the interface: every panel addressed by /wkx/<service>/<env>, Terraform plan symbols for state, and hover-to-highlight on every token of the ubiquitous language. Superseded by option 4.

  • 4 · Namespace · dense, interactive, light and dark

    The namespace by night or by day, following the system theme with a manual override: the console layout and status lights of Harbour Watch, speaking the path grammar. Panels addressed by /wkx/<service>/<env>, plan symbols for state, and hover-to-highlight on every token of the ubiquitous language.
Static HTML and JS only, no build step. Open any option directly, or serve the folder with python3 -m http.server. Content reflects the platform as at M5, with M9 and M10 surfaces shown as honest empty states.
